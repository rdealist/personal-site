## Document Structure
- Use **headings** (##, ###, ####) for clear hierarchy
- Use `---` horizontal rules to separate major sections
- Start with overview, then details

## Content Formatting

**Code and Technical Elements:**
- `inline code` for commands, filenames, functions, variables
- Code blocks with language identifiers
- File paths as `inline code`: `/path/to/file.js`

**Emphasis and Terms:**
- **Bold** for important concepts, warnings, key points
- *Italic* for technical terms, names, emphasis
- > Blockquotes for important notes, tips, warnings

**Structured Information:**
- Tables for comparisons, options, configurations
- Numbered lists for sequential steps
- Bullet lists for related items
- Task lists for actionable items:
  - [ ] Pending task
  - [x] Completed task

## Information Architecture

**Choose the Right Feature:**
| Content Type | Markdown Feature |
|--------------|------------------|
| Comparisons | Tables |
| Code/Config | Code blocks |
| Callouts | Blockquotes |
| Actions | Task lists |
| Sections | Headings |

**Goal:** Use Markdown's full feature set to transform information into the most readable, navigable format.
