## 🚀 笔记发布流程

### 1. 添加新笔记

在 `ai-glossary/` 目录下创建新的 Markdown 文件:

```bash
# 创建新笔记
touch ai-glossary/新主题名称.md
```

### 2. 笔记格式规范

每个笔记文件应遵循以下Markdown格式:

```markdown
# 主题名称

## 1. 第一个问题或概念

**定义**: 简短定义

**详细说明**:
- 要点1
- 要点2

**示例**:
\`\`\`
代码或示例
\`\`\`

---

## 2. 第二个问题或概念

...
```

### 3. 可选的Front Matter

您可以在文件开头添加元数据(可选):

```markdown
---
order: 1
description: "这是一篇关于...的笔记"
---

# 主题名称
...
```

### 4. 自动发布

- ✅ 文件保存后会自动被系统检测
- ✅ 笔记会自动出现在 `/notes` 页面
- ✅ 自动生成URL slug(基于文件名)
- ✅ 自动提取描述(来自front matter或内容开头)

## 🔧 技术实现

### 核心文件

1. **`src/lib/notes.ts`** - 笔记数据加载器
   - `getAllNotes()` - 获取所有笔记
   - `getNoteBySlug()` - 根据slug获取单个笔记
   - `searchNotes()` - 搜索笔记(未来功能)

2. **`src/app/[locale]/notes/page.tsx`** - 笔记列表页
   - 展示所有笔记的网格视图
   - 分类导航

3. **`src/app/[locale]/notes/[slug]/page.tsx`** - 笔记详情页
   - Markdown渲染
   - 响应式设计
   - 分享功能

4. **`src/components/sections/notes-preview-section.tsx`** - 首页笔记预览
   - 精选笔记主题
   - 快速导航

## 📝 笔记编写建议

### 结构化内容

- 使用清晰的标题层级 (`#`, `##`, `###`)
- 用表格对比不同概念
- 使用代码块展示示例
- 添加引用和参考链接

### 视觉元素

- ✅ 支持表格
- ✅ 支持代码高亮
- ✅ 支持引用块
- ✅ 支持列表(有序/无序)
- ✅ 支持emoji

### 示例模板

```markdown

**核心定义**: 一句话说明

**关键特征**:
| 特征 | 说明 |
|------|------|
| 特征1 | 说明1 |
| 特征2 | 说明2 |

---

## XX的应用场景

1. **场景一**: 描述
2. **场景二**: 描述

---

## 实践示例

\`\`\`python
# 代码示例
def example():
    pass
\`\`\`

---

## 相关资源

- [官方文档](链接)
- [相关论文](链接)
```

## 🔄 更新流程

### 修改现有笔记

1. 直接编辑 `ai-glossary/` 目录下的对应文件
2. 保存文件
3. 刷新网站查看更新

### 删除笔记

1. 删除 `ai-glossary/` 目录下的对应文件
2. 笔记会自动从网站移除

### 重新排序

在文件的 front matter 中设置 `order` 字段:

```markdown
---
order: 1
---
```

数字越小,排序越靠前。

## 🎨 样式定制

笔记详情页使用了 Tailwind Typography,支持以下定制:

- 代码块有语法高亮
- 表格有边框和斑马纹
- 引用块有特殊样式
- 标题有不同层级样式
